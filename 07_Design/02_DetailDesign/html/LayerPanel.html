LayerPanel.tsx - 詳細設計

← トップページへ戻る

1. ファイル概要

ファイル名LayerPanel.tsx
パッケージsrc/components/
役割レイヤー一覧の表示、レイヤーの追加・削除・表示切替・スタイル編集を行うUIコンポーネント
依存関係
  • react (hooks: useMemo, useRef, useState)
  • types/layer.ts (Layer, LayerStyle)

2. 処理詳細

2.1 主要な責務

  • レイヤーの一覧表示とアクティブレイヤーのハイライト
  • 新規レイヤーの追加フォーム
  • 各レイヤーの名前・スタイル（線色・塗り色・不透明度）の編集
  • レイヤーの表示/非表示の切り替え
  • レイヤーの削除
  • レイヤー一覧のJSON保存・読み込み
  • フィーチャ数の集計表示

2.2 Props仕様

プロパティ名型説明
layersLayer[]表示するレイヤーの配列
activeLayerIdstring | null現在アクティブなレイヤーのID
onSelectLayer(layerId: string) => voidレイヤー選択時のコールバック
onAddLayer(name: string) => voidレイヤー追加時のコールバック
onDeleteLayer(layerId: string) => voidレイヤー削除時のコールバック
onToggleLayer(layerId: string) => voidレイヤー表示切替時のコールバック
onUpdateLayer(layerId: string, updates: object) => voidレイヤー更新時のコールバック
onSaveLayerList() => voidレイヤー一覧保存時のコールバック
onLoadLayerList(file: File) => voidレイヤー一覧読み込み時のコールバック

2.3 状態管理

状態名型説明
newLayerNamestring新規レイヤー名の入力値
totalFeaturesnumber (computed)全レイヤーのフィーチャ総数（useMemoで算出）

2.4 主要メソッド

メソッド名概要
handleAddLayer入力フィールドの値をトリムして親コンポーネントのonAddLayerを呼び出し、入力をクリア
handleLayerFileChangeファイル選択時に親コンポーネントのonLoadLayerListを呼び出し、入力値をリセット
triggerLayerImport非表示のファイル入力要素をクリックしてファイル選択ダイアログを表示

3. 入出力仕様

3.1 入力

項目型説明
レイヤー配列Layer[]親コンポーネントから渡されるレイヤーデータ
ユーザー入力string, Event新規レイヤー名、スタイル変更、ファイル選択など

3.2 出力

項目型説明
コールバック呼び出しvoid親コンポーネントのハンドラー関数を呼び出し、状態変更を依頼
UIレンダリングReactElementレイヤーツリーUIを返す

4. 例外・バリデーション仕様

条件処理
空のレイヤー名トリム後に親へ渡すため、空文字でもonAddLayerが呼ばれる（親側でバリデーション想定）
ファイル未選択files?.[0]がundefinedの場合は何もしない

備考: このコンポーネントはUIの責務に集中しており、ビジネスロジックのバリデーションは親コンポーネント（MapCanvas）で行われます。
