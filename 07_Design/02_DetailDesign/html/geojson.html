1. ファイル概要

ファイル名geojson.ts
パッケージsrc/utils/
役割Google Maps図形とGeoJSON Feature間の相互変換ユーティリティ
依存関係
  • geojson型 (Feature, FeatureCollection, Geometry, Position)
  • google.maps型 (各種図形クラス)

2. 処理詳細

2.1 定数

定数名値説明
EARTH_RADIUS_IN_METERS6,378,137地球の半径（メートル）。円のポリゴン近似計算に使用

2.2 公開関数

関数名説明
convertOverlayToFeature(event: OverlayCompleteEvent): Feature | nullGoogle Maps Drawing Managerで作成された図形（Marker, Polyline, Polygon, Rectangle, Circle）をGeoJSON Featureに変換。変換できない場合はnullを返す
normalizeGeoJson(raw: unknown): Feature | Feature[] | null不明な形式のJSONをGeoJSON形式に正規化。FeatureCollection, Feature, Feature配列を解析し、統一的にFeatureまたはFeature配列として返す

2.3 内部ヘルパー関数

関数名説明
createFeatureId(): stringcrypto.randomUUID()またはタイムスタンプベースでユニークなFeature IDを生成
pathToLinearRing(path: MVCArray<LatLng>): Position[]Google MapsのMVCArray座標をGeoJSONのLinearRing（閉じた座標配列）に変換。最初と最後の座標が異なる場合は自動で閉じる
circleToPolygon(circle: Circle, segments: number): Position[][]円を指定セグメント数のポリゴンに近似変換（デフォルト48セグメント）
rectangleToPolygon(rectangle: Rectangle): Position[][]矩形の境界をポリゴンの座標配列に変換
polylineToCoordinates(polyline: Polyline): Position[]PolylineをLineString座標配列に変換
polygonToCoordinates(polygon: Polygon): Position[][]Polygonの外側リングと内側リング（穴）をGeoJSON座標配列に変換

3. 入出力仕様

3.1 convertOverlayToFeature

入力

パラメータ名型説明
eventgoogle.maps.drawing.OverlayCompleteEventDrawing Managerの図形完成イベント。event.type と event.overlay を含む

出力

型説明
Feature | null変換成功時はGeoJSON Feature（id, geometry, propertiesを含む）、失敗時はnull

サポートする図形タイプ

  • MARKER → Point
  • POLYLINE → LineString
  • POLYGON → Polygon
  • RECTANGLE → Polygon
  • CIRCLE → Polygon（近似）

3.2 normalizeGeoJson

入力

パラメータ名型説明
rawunknown任意のJSON形式データ

出力

型説明
Feature | Feature[] | nullFeatureCollectionの場合はfeatures配列、単一Featureの場合はそのまま、配列の場合は配列として返す。不正な形式の場合はnull

4. 例外・バリデーション仕様

条件処理
overlayがnull/undefinednullを返す
未サポートの図形タイプnullを返す
座標取得失敗（例: position/bounds/path が null）nullを返す
座標数不足（例: LineString が2点未満、Polygon が4点未満）nullを返す
rawがnullまたはオブジェクトでないnullを返す
typeプロパティがない、またはFeature/FeatureCollectionでない配列の場合はそのまま返し、それ以外はnull
